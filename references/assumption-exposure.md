# Assumption Exposure

Use this route when the user's conclusion depends on premises they have not yet stated.

## Goal

Reveal what the claim needs in order to stand.

## What counts as an assumption here

An assumption is not merely any belief.
It is the support beam the claim is standing on without acknowledging it.

## Common modern patterns

- "This is obviously better"
- "Everyone wants this"
- "Growth means health"
- "Freedom means fewer constraints"
- "If it works, it is good"
- "If AI can do it, humans need not"

## Working method

1. State the visible claim
2. Ask what must be true for that claim to hold
3. Separate factual assumptions from evaluative assumptions
4. Test whether the user would still hold the claim if one assumption failed
5. End with the most load-bearing assumption named clearly

## Useful question forms

- "What has to be true for that conclusion to follow?"
- "What are you taking for granted here?"
- "If that assumption proved false, would the conclusion survive?"
- "Are you assuming a fact, or a value judgment?"
- "What standard are you using without saying it?"

## Distinctions to keep

### Factual assumption

Something about how the world works.

### Evaluative assumption

Something about what counts as good, bad, worth doing, or worth avoiding.

### Identity assumption

Something about the kind of person or organization one is trying to be.

### Temporal assumption

Something about what matters now versus later.

## Failure modes

- confusing the user's fear with the user's premise
- turning every plan into a pathology
- calling everything an assumption and nothing a conclusion
- refusing to name the assumption directly

## Example use

Claim:
"I need to start a company to be free."

Possible hidden assumptions:

- a boss is the main source of unfreedom
- self-employment is less constraining than employment
- financial insecurity is more tolerable than subordination
- the user values autonomy above stability
- freedom is defined negatively rather than positively

## Ending shape

Prefer to end with:

- the visible claim
- the hidden assumption now made explicit
- the next test that assumption should face
